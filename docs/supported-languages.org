* ‚ö†Ô∏è Overview

Scimax VS Code supports a wide range of programming languages for source block execution through two primary mechanisms:

1. *Native Execution* - Direct language interpreters (Python, JavaScript, Shell)
2. *Jupyter Kernels* - Any language with a Jupyter kernel installed

This document provides a comprehensive reference for all supported languages, including syntax examples, execution modes, session support, and required dependencies.

* ‚ö†Ô∏è Understanding Execution Modes

** ‚ö†Ô∏è Native Execution

Native executors run code directly using installed command-line interpreters. They are:
- Fast and lightweight
- No additional setup beyond language installation
- Limited session support (state doesn't persist between blocks)

Supported natively: Python, JavaScript/Node.js, Shell/Bash

** ‚ö†Ô∏è Jupyter Execution

Jupyter executors use Jupyter kernels for code execution. They provide:
- Full session support (variables persist between blocks)
- Rich output (images, plots, formatted data)
- Interactive features
- Support for many languages

To use Jupyter for any language:
1. *Explicit syntax*: Use jupyter-<language> (e.g., jupyter-python)
2. *Auto-detection*: Some languages automatically use Jupyter if available

** ‚ö†Ô∏è Determining Which Executor Runs

import (
    "fmt"
    "sync"
)

func worker(id int, wg *sync.WaitGroup) {
    defer wg.Done()
    fmt.Printf("Worker %d starting\n", id)
    fmt.Printf("Worker %d done\n", id)
}

func main() {
    var wg sync.WaitGroup
    for i := 1; i <= 3; i++ {
        wg.Add(1)
        go worker(i, &wg)
    }
    wg.Wait()
}

* ‚ö†Ô∏è Troubleshooting

** ‚ö†Ô∏è Checking Available Executors

Use the command Scimax: Check Available Executors to see which language executors are available.

** ‚ö†Ô∏è Common Issues

*** ‚ö†Ô∏è "No executor available for language: X"

*Cause:* The language interpreter or Jupyter kernel is not installed.

*Solution:*
1. For native languages: Install the language (python3, node, etc.)
2. For Jupyter languages: Install the appropriate kernel
3. Verify installation: jupyter kernelspec list

*** ‚ö†Ô∏è "Jupyter kernel not found"

*Cause:* Jupyter is not installed or kernel is not registered.

*Solution:*
#+BEGIN_SRC bash
# Install Jupyter
pip install jupyter

# Install specific kernel (example for Python)
python -m ipykernel install --user

# Verify kernels
jupyter kernelspec list
#+END_SRC

*** ‚ö†Ô∏è Session state not persisting

*Cause:* Using native executor instead of Jupyter, or session name not specified.

*Solution:*
- Use jupyter-<language> syntax
- Add :session <name> header argument
- Ensure all blocks use the same session name

*** ‚ö†Ô∏è Code executes but no output appears

*Cause:* Results handling configuration or silent mode enabled.

*Solution:*
- Check :results header (use :results output)
- Ensure :eval is not set to no
- Add explicit print statements
- Check if :results silent is set

*** ‚ö†Ô∏è Import/Module not found errors

*Cause:* Wrong Python environment or missing packages.

*Solution:*
#+BEGIN_SRC bash
# Check which Python is being used
which python3

# Install missing package
pip install package-name

# Use specific environment
conda activate myenv
#+END_SRC

* ‚ö†Ô∏è Best Practices

** ‚ö†Ô∏è Language Selection
- Use native executors for simple, stateless scripts
- Use Jupyter for interactive work requiring sessions
- Use explicit jupyter-<language> when you need Jupyter features

** ‚ö†Ô∏è Session Management
- Name sessions descriptively (:session *data-analysis*)
- Use separate sessions for independent workflows
- Clear sessions when starting fresh work

** ‚ö†Ô∏è Code Organization
- One concept per block for clarity
- Use named blocks (#+NAME:) for reusable code
- Document complex blocks with comments

** ‚ö†Ô∏è Performance
- Cache expensive computations (:cache yes)
- Use appropriate result types (:results output vs value)
- Close unused Jupyter sessions

** ‚ö†Ô∏è Reproducibility
- Specify explicit :dir for file operations
- Use :session consistently in related blocks
- Document dependencies and versions

* ‚ö†Ô∏è Quick Reference

** ‚ö†Ô∏è Execute Commands
- Ctrl+Enter / Cmd+Enter: Execute current block
- Shift+Enter: Execute and move to next
- Ctrl+C Ctrl+C: Execute (Emacs style)

** ‚ö†Ô∏è Session Control
- :session <name>: Use named session
- :session none: Disable session
- Sessions persist until VS Code restart or manual close

** ‚ö†Ô∏è Result Handling
- :results output: Capture stdout
- :results value: Return last value
- :results silent: No results
- :results file: Save to file

** ‚ö†Ô∏è Jupyter Features
- Rich output (plots, tables, HTML)
- Tab completion
- Object inspection
- Persistent state
