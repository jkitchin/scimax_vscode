* ‚úÖ LaTeX Navigation and Structure
CLOSED: [2026-01-19 Mon 09:24]

Scimax VS Code provides comprehensive navigation, structure editing, and tooltips for LaTeX documents. These features mirror the org-mode heading navigation and structure editing commands, adapted for LaTeX section commands and environments.

** ‚úÖ Overview
CLOSED: [2026-01-19 Mon 09:24]

LaTeX support in Scimax VS Code includes:

1. *Document Outline* - Hierarchical view of sections, environments, and labels
2. *Section Navigation* - Move between sections at any level
3. *Structure Editing* - Promote/demote, move, and manipulate sections
4. *Environment Commands* - Select, change, wrap, and unwrap environments
5. *Hover Tooltips* - Information about references, citations, packages, and commands
6. *Speed Commands* - Single-key shortcuts at section start
7. *Folding* - Collapse and expand sections and environments
8. *Go to Definition* - Jump to labels, citations, commands, and included files
9. *Find References* - Find all uses of a label or citation key
10. *Auto-Completion* - Smart completions for labels, citations, environments, packages

* ‚úÖ Document Outline
CLOSED: [2026-01-19 Mon 09:24]

The outline view (VS Code's built-in outline panel `Cmd/C-Shift-o') shows the structure of your LaTeX document:

- *Sections* (part, chapter, section, subsection, etc.)
- *Labeled Environments* (figures, tables, equations with labels)
- *Labels* for cross-referencing

The outline uses hierarchical nesting to show the document structure. Click any item to jump to it.

** ‚úÖ Section Hierarchy
CLOSED: [2026-01-19 Mon 09:26]

LaTeX sections are organized hierarchically:

| Level | Command         | Symbol Kind |
|-------+-----------------+-------------|
|     0 | \part           | Module      |
|     1 | \chapter        | Class       |
|     2 | \section        | Method      |
|     3 | \subsection     | Function    |
|     4 | \subsubsection  | Field       |
|     5 | \paragraph      | Property    |
|     6 | \subparagraph   | Property    |

Starred variants (e.g., =\section*{...}=) are also recognized and marked in the outline.

** ‚úÖ Next/Previous Section
CLOSED: [2026-01-19 Mon 09:26]

Move through all sections in document order:

#+BEGIN_SRC latex
\section{Introduction}
Content here...

\subsection{Background}    % <- Current position

\subsection{Motivation}    % <- Next section

\section{Methods}          % <- Another next section
#+END_SRC

=nextSection= moves forward through sections regardless of level. =previousSection= moves backward.

** ‚úÖ Sibling Navigation
CLOSED: [2026-01-19 Mon 09:26]

Move between sections at the same level:

#+BEGIN_SRC latex
\section{Introduction}

\subsection{Background}    % <- Current position
\subsection{Motivation}    % <- Next sibling (same level)
\subsection{Goals}         % <- Another sibling

\section{Methods}          % <- NOT a sibling (different level)
#+END_SRC

=nextSiblingSection= moves to the next section at the same hierarchical level.
=previousSiblingSection= moves backward.

** ‚úÖ Parent Section
CLOSED: [2026-01-19 Mon 09:26]

Jump to the containing (parent) section:

#+BEGIN_SRC latex
\section{Methods}            % <- Jump here (parent)
\subsection{Data Collection}
\subsubsection{Sampling}     % <- From here
#+END_SRC

** ‚úÖ Promote/Demote
CLOSED: [2026-01-19 Mon 09:27]

Change the level of a section command:

#+BEGIN_SRC latex
% Before promote:
\subsection{Methods}

% After promote (Alt+Left):
\section{Methods}

% Before demote:
\section{Results}

% After demote (Alt+Right):
\subsection{Results}
#+END_SRC

** ‚úÖ Promote/Demote Subtree
CLOSED: [2026-01-19 Mon 09:27]

#+BEGIN_SRC latex
% Before promoteSubtree on "Methods":
\subsection{Methods}
\subsubsection{Data}
\subsubsection{Analysis}

% After promoteSubtree (Alt+Shift+Left):
\section{Methods}
\subsection{Data}
\subsection{Analysis}
#+END_SRC

** ‚úÖ Move Section Up/Down
CLOSED: [2026-01-19 Mon 09:27]

Swap a section with its previous or next sibling:

#+BEGIN_SRC latex
% Before moveSectionDown on "Introduction":
\section{Introduction}
Content A...

\section{Methods}
Content B...

% After moveSectionDown (Alt+Down):
\section{Methods}
Content B...

\section{Introduction}
Content A...
#+END_SRC

The entire section subtree (including all nested content) is moved.

* ‚úÖ Speed Commands
CLOSED: [2026-01-19 Mon 09:29]

At the beginning of a section line, single keys trigger commands:

| Key | Action                 | Key | Action             |
|-----+------------------------+-----+--------------------|
| n   | Next section           | <   | Promote section    |
| p   | Previous section       | >   | Demote section     |
| f   | Next sibling           | U   | Move section up    |
| b   | Previous sibling       | D   | Move section down  |
| u   | Parent section         | m   | Mark section       |
| j   | Jump to section        | k   | Kill section       |
| J   | Jump to label          | c   | Clone section      |
| g   | First section          | i   | Insert section     |
| G   | Last section           | I   | Insert subsection  |
| Tab | Toggle fold            | N   | Narrow to section  |
| S-Tab | Cycle global fold    | W   | Widen              |
| ?   | Show speed commands    |     |                    |

Speed commands only activate when cursor is at or before the backslash of a section command.

** ‚úÖ Using Speed Commands
CLOSED: [2026-01-19 Mon 09:29]

1. Place cursor at the start of a section line (column 0 or before =\=)
2. Press a single key
3. The corresponding action executes

Example:
#+BEGIN_SRC latex
\section{Introduction}   % <- Place cursor here, press 'n' to go to next section
  ^
  |__ Cursor position
#+END_SRC

** ‚úÖ Section Folding
CLOSED: [2026-01-19 Mon 09:29]

#+BEGIN_SRC latex

#+END_SRC

** ‚úÖ Compilation
CLOSED: [2026-01-19 Mon 09:31]

The =compile= command runs the configured LaTeX compiler (default: pdflatex):

Configure the compiler via =scimax.latex.compiler= setting:
- =pdflatex= (default)
- =xelatex=
- =lualatex=

* ‚ö†Ô∏è Multi-File Project Support

Projects with multiple files (using =\input= or =\include=) are fully supported:

- *Master document detection*: Automatically finds the main .tex file
- *Cross-file labels*: Complete and navigate labels defined in any project file
- *Shared bibliography*: Finds .bib files referenced anywhere in project
- *Project-wide search*: Find References searches all included files

The master document is detected by looking for =\documentclass=.

** ‚ö†Ô∏è Project Structure

#+BEGIN_SRC text

=latexindent= must be installed:

- *Linux*: =apt install texlive-extra-utils= or =cpanm YAML::Tiny File::HomeDir Unicode::GCString=

* ‚úÖ LaTeX-Aware Spell Checking
CLOSED: [2026-01-19 Mon 09:12]

The extension provides infrastructure for LaTeX-aware spell checking that
skips commands, math, and comments.

** ‚úÖ Regions Excluded from Spell Check
CLOSED: [2026-01-19 Mon 09:12]

| Pattern                          | Example                      |
|----------------------------------+------------------------------|
| Commands with arguments          | =\textbf{word}=              |
| Inline math                      | =$x^2 + y^2$=                |
| Display math                     | =$$..$$= and =\[..\]=        |
| Environments                     | =\begin{...}...\end{...}=    |
| Comments                         | =% comment text=             |
