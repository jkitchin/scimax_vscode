* ✅ Project Management
CLOSED: [2026-07-02 Thu 08:07]

Building on [[file:03.5-task-dependencies.org][task dependencies]], Scimax turns a set of org TODO tasks into
a small project-management view: a **task table**, a **Gantt chart**, a
**dependency graph**, and a **people database** with assignee autocomplete.
Everything is driven off metadata you already put on headings —
=SCHEDULED=/=DEADLINE=, =EFFORT=, priority, =:DEPENDS:=, and =:ASSIGNEE:=.

** ✅ The data a task carries
CLOSED: [2026-07-02 Thu 08:07]

#+begin_src org
,* Project X
  :PROPERTIES:
  :ID: proj-x
  :ASSIGNEE: jrk          ← whole subtree assigned to jrk (inherited)
  :END:
,** TODO Run analysis
   SCHEDULED: <2026-07-01 Wed>
   :PROPERTIES:
   :ID: analysis
   :EFFORT: 2d
   :END:
,** NEXT Make figures                      :@ana:   ← @tag also assigns
   :PROPERTIES:
   :ID: figures
   :EFFORT: 4h
   :DEPENDS: id:analysis
   :END:
,** TODO [#A] Write paper
   DEADLINE: <2026-07-20 Mon>
   :PROPERTIES:
   :ID: paper
   :EFFORT: 1d
   :DEPENDS: id:analysis id:figures
   :END:
#+end_src

- =EFFORT= accepts =2d=, =4h=, =0:30=, =90m= (a day = 8 hours).
- =:ASSIGNEE:= holds one or more handles; it is *inherited* from ancestor
  subtrees, and =@name= tags count as assignees too.

** ✅ Task table
CLOSED: [2026-07-02 Thu 08:07]

Insert a table with [[cmd:scimax.org.insertProjectTable]], or write the dynamic
block yourself and refresh it with =C-c C-c=:

#+begin_src org
,#+BEGIN: project-table :columns task,todo,priority,assignee,deadline,effort,blocked
,#+END:
#+end_src

Parameters:

| Param             | Meaning                                                                 |
|-------------------+-------------------------------------------------------------------------|
| =:columns=        | Any of task, todo, priority, assignee, scheduled, deadline, effort, blocked, deps |
| =:id=             | Scope to the subtree with this heading =:ID:= (whole file if omitted)   |
| =:maxlevel=       | Only include headings up to this level                                  |
| =:match=          | Only tasks with these tags, e.g. =+urgent=                              |
| =:groupby=        | =assignee= or =state= — adds a grouping column                          |
| =:include_non_todo= | =t= to include headings without a TODO keyword                       |

The =blocked= column shows 🔒 when a same-file dependency is not yet done.

** ✅ Gantt chart
CLOSED: [2026-07-02 Thu 08:07]

Insert with [[cmd:scimax.org.insertGantt]] (or write the block and =C-c C-c=):

#+begin_src org
,#+BEGIN: gantt :title Project X :sections assignee
,#+END:
#+end_src

This generates a [[https://mermaid.js.org/syntax/gantt.html][Mermaid]] =gantt= diagram inside a =#+begin_src mermaid= block,
which you can render to an image with =C-c C-c= (Mermaid is a supported diagram
language). Mapping:

- =SCHEDULED= → task start; =:DEPENDS:= → Mermaid =after=; =EFFORT= → duration
  (rounded up to working days).
- A task with only a =DEADLINE= (no effort/schedule) becomes a *milestone*.
- =DONE=/=CANCELLED= → =done=; a started, scheduled task → =active=; priority
  =[#A]= → =crit=.

Parameters: =:title=, =:id=, =:maxlevel=, =:match=, =:sections= (=assignee=,
=parent=, or =none=), =:crit_priority= (default =A=).

** ✅ Dependency graph
CLOSED: [2026-07-02 Thu 08:07]

[[cmd:scimax.org.showTaskGraph]] opens an interactive graph (nodes = tasks,
arrows = "depends on"), colored by status:

- 🔵 ready · 🔴 blocked · 🟢 done · ⚪ unresolved id

It is seeded from the active file's tasks and expands over the dependency edges
to show the connected component. Click a node to jump to that task.

** ✅ People and assignments
CLOSED: [2026-07-02 Thu 08:07]

A *person* is any heading tagged =:person:=, with contact properties:

#+begin_src org
,* John Kitchin                              :person:
  :PROPERTIES:
  :ID: person-jrk
  :EMAIL: jkitchin@andrew.cmu.edu
  :NICK: jrk
  :ROLE: PI
  :END:
#+end_src

- **Autocomplete**: while editing an =:ASSIGNEE:= value, completion suggests
  known people (name + email) and inserts their handle (=:NICK:=, or a slug of
  the name).
- **Hover**: hovering an assignee handle shows the person's name, email, and role.
- **Capture**: [[cmd:scimax.org.newPerson]] prompts for name/email/nick/role and
  appends a =:person:= heading to your people file
  (=scimax.org.peopleFile=, default ={scimax.directory}/people.org=).

Assignees feed the table's =assignee= column, the Gantt's per-person swimlanes
(=:sections assignee=), and grouping (=:groupby assignee=).

** ✅ Configuration
CLOSED: [2026-07-02 Thu 08:07]

| Setting                    | Default | Description                                   |
|----------------------------+---------+-----------------------------------------------|
| =scimax.org.peopleFile=    | (empty) | Where "New Person" appends; empty ⇒ people.org |

Task-dependency settings (blocking, promotion, indicators) are documented in
[[file:03.5-task-dependencies.org][Task Dependencies]].

** ✅ Notes and limitations
CLOSED: [2026-07-02 Thu 08:07]

- The table and Gantt are computed from a *single file* (or a subtree via
  =:id=), so keep a project's tasks together. Cross-file dependencies still
  drive blocking and the dependency graph.
- The "blocked" column and Gantt ordering resolve dependencies within the block's
  scope; cross-file dependency state is shown in the dependency graph instead.
- Rendering the Gantt to an image uses the existing Mermaid source-block support
  (=C-c C-c=).
