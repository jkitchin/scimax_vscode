* ‚ö†Ô∏è Getting Started

This guide will help you install Scimax VS Code and get started with its core
features. By the end, you'll be able to create structured documents, execute
code, and navigate efficiently.

* ‚ö†Ô∏è System Requirements

** ‚ö†Ô∏è VS Code Version

Scimax VS Code requires Visual Studio Code version *1.85 or later*.

** ‚ö†Ô∏è Optional Dependencies

While Scimax VS Code works out of the box, some features require additional
software:

*** ‚ö†Ô∏è For Code Execution

| Language   | Requirement                                    |
|------------+------------------------------------------------|
| Python     | Python 3.x installed and in PATH               |
| JavaScript | Node.js installed                              |
| Shell      | Bash or compatible shell                       |
| R          | R installed with Rscript in PATH               |
| Julia      | Julia installed and in PATH                    |

*** ‚ö†Ô∏è For Jupyter Integration

- Jupyter installed (pip install jupyter)
- Jupyter kernels for desired languages

*** ‚ö†Ô∏è For LaTeX/PDF Export

- TeX distribution (TeX Live, MiKTeX, or MacTeX)
- pdflatex, xelatex, or lualatex compiler

*** ‚ö†Ô∏è For LaTeX Equation Preview

- LaTeX distribution with dvipng or dvisvgm

* ‚ö†Ô∏è Installation

** ‚ö†Ô∏è From VS Code Marketplace

1. Open VS Code
2. Press Ctrl+Shift+X (Cmd+Shift+X on Mac) to open the Extensions view
3. Search for "Scimax"
4. Click *Install*

** ‚ö†Ô∏è From VSIX File

1. Download the .vsix file
2. Open VS Code
3. Press Ctrl+Shift+X (Cmd+Shift+X on Mac) to open the Extensions view
4. Click the ... menu (top right)
5. Select "Install from VSIX..."
6. Choose the downloaded file

** ‚ö†Ô∏è Verify Installation

After installation:

1. Open the Command Palette (Ctrl+Shift+P / Cmd+Shift+P on Mac)
2. Type "Scimax"
3. You should see Scimax commands listed

* ‚ö†Ô∏è First Steps

** ‚ö†Ô∏è Opening an Org File

Org files use the .org extension. To create your first file:

1. Press Ctrl+N (Cmd+N on Mac) to create a new file
2. Save it with Ctrl+S (Cmd+S on Mac) and give it a .org extension (e.g., notes.org)
3. The file will automatically use org-mode syntax highlighting

** ‚ö†Ô∏è Basic Navigation

*** ‚ö†Ô∏è Folding

- Press Tab on a heading to cycle its visibility (folded/children/subtree)
- Press Shift+Tab anywhere to cycle global visibility

*** ‚ö†Ô∏è Moving Between Headings

| Key            | Action                    |
|----------------+---------------------------|
| Ctrl+C Ctrl+J  | Jump to any heading       |
| Alt+Up         | Move heading up           |
| Alt+Down       | Move heading down         |
| Alt+Left       | Promote heading (‚Üêlevel)  |
| Alt+Right      | Demote heading (‚Üílevel)   |

*** ‚ö†Ô∏è Executing Code

1. Place your cursor inside a source block
2. Press Ctrl+Enter (Cmd+Enter on Mac) or Ctrl+C Ctrl+C
3. Results appear below the block

* ‚ö†Ô∏è Understanding the Interface

** ‚ö†Ô∏è Outline View

The Outline view in VS Code's sidebar shows your document structure:

1. Click the Outline icon in the sidebar (or press Ctrl+Shift+O / Cmd+Shift+O on Mac)
2. See all headings hierarchically
3. Click a heading to jump to it

** ‚ö†Ô∏è Tree Views

Scimax VS Code adds several tree views to the Explorer sidebar:

*** ‚ö†Ô∏è Journal Tree

Shows your journal entries organized by date. Access today's entry with
Ctrl+Shift+J (Cmd+Shift+J on Mac).

*** ‚ö†Ô∏è Projects Tree

Shows your registered projects for quick navigation.

*** ‚ö†Ô∏è Agenda Tree

Shows TODO items, deadlines, and scheduled tasks across your files.

** ‚ö†Ô∏è Status Bar

The status bar shows:

- Current journal date (when in journal files)
- Kernel status (when using Jupyter)

** ‚ö†Ô∏è Hydra Menus

Hydra menus provide discoverable access to commands:

- Ctrl+C M - Open main menu
- Ctrl+C Ctrl+M - Open context-sensitive menu
- Ctrl+Alt+V (Ctrl+Cmd+V on Mac) - Open database/search menu

Menus show available commands with their keybindings. Press a key to execute
a command or Escape to close the menu.

* ‚ö†Ô∏è Recommended Settings

Open VS Code settings (Ctrl+, / Cmd+, on Mac) and search for "scimax" to see all options.

** ‚ö†Ô∏è Essential Settings

#+BEGIN_SRC json
{
  // Enable speed commands (single-key at heading start)
  "scimax.speedCommands.enabled": true,

  // Set your journal directory
  "scimax.journal.directory": "~/Documents/journal",

  // Configure bibliography files for citations
  "scimax.ref.bibliographyFiles": ["~/references.bib"],

  // Enable image overlays
  "scimax.imageOverlays.enabled": true,

  // Auto-index files for search
  "scimax.db.autoIndex": true
}
#+END_SRC

* ‚ö†Ô∏è Migrating from Emacs

If you're coming from Emacs Org-mode or Scimax, here are key differences:

** ‚ö†Ô∏è Similar Features

| Emacs                | VS Code                          |
|----------------------+----------------------------------|
| C-c C-c              | Ctrl+C Ctrl+C                    |
| C-c C-t              | Ctrl+C Ctrl+T                    |
| C-c C-e              | Ctrl+C Ctrl+E                    |
| TAB on heading       | Tab on heading                   |
| S-TAB                | Shift+Tab                        |
| M-RET                | Ctrl+Enter (at heading)          |
| M-left/right         | Alt+Left/Right                   |
| M-up/down            | Alt+Up/Down                      |
| Speed commands       | Same keys (when enabled)         |

** ‚ö†Ô∏è Different Behaviors

- *Completion*: Uses VS Code's completion system, triggered automatically
- *Folding*: Uses VS Code's native folding with org-specific ranges
- *Buffer management*: VS Code tabs instead of Emacs buffers
- *Customization*: JSON settings instead of Emacs Lisp

** ‚ö†Ô∏è Missing Features

Some advanced Emacs features may not be available:

- Org-mode's full capture/refile system (simplified version available)
- Column view
- Some specialized agenda views
- Custom link types (limited set supported)

* ‚ö†Ô∏è Troubleshooting

** ‚ö†Ô∏è Code Blocks Not Executing

1. Check that the language is installed (e.g., Python, Node.js)
2. Verify the language is in your PATH
3. Check the Output panel (Ctrl+Shift+U / Cmd+Shift+U on Mac) for errors
4. Run scimax.org.checkExecutors to verify available executors

** ‚ö†Ô∏è LaTeX Preview Not Working

1. Ensure LaTeX is installed (latex --version in terminal)
2. Check that dvipng or dvisvgm is available
3. Run scimax.checkLatexTools to verify installation

** ‚ö†Ô∏è Search Not Finding Results

1. Run scimax.db.reindex to rebuild the search index
2. Check that files aren't in excluded patterns
3. Verify the search scope (scimax.db.setScope)

** ‚ö†Ô∏è Journal Not Opening

1. Check that scimax.journal.directory is set correctly
2. Ensure the directory exists and is writable
3. Try creating the directory manually if needed
