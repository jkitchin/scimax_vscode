Scimax VS Code Documentation Plan

Documentation Team

2026-01-13

⚠️ Overview

This document outlines the comprehensive documentation plan for Scimax VS Code, a scientific computing extension for Visual Studio Code inspired by Emacs Scimax.

The documentation will be written entirely in org-mode format and will cover all features, keybindings, configuration options, and usage patterns.

⚠️ Documentation Goals

  1. Complete coverage - Document every feature the extension provides

  2. User-friendly - Progressive disclosure from basics to advanced

  3. Task-oriented - Focus on what users want to accomplish

  4. Reference material - Quick lookup for keybindings and commands

  5. Examples - Show practical usage patterns

⚠️ Target Audience

  • Scientists and researchers doing literate programming

  • Note-takers and knowledge workers

  • Developers using org-mode for documentation

  • Users migrating from Emacs org-mode/Scimax

⚠️ Document Standards

⚠️ File Header Template

Every documentation file should begin with:

,#+TITLE: [Document Title]
,#+AUTHOR: Scimax VS Code Team
,#+DATE: [Date]
,#+STARTUP: overview
,#+OPTIONS: toc:2 num:t H:4
,#+PROPERTY: header-args :eval never

,* Introduction
[Brief overview of the topic]

⚠️ Section Structure

  • Each major section should have an introduction

  • Include practical examples where appropriate

  • Show keybindings in tables or inline with key

  • Use command-name for VS Code commands

  • Cross-reference related sections with links

⚠️ Example Formatting

Code examples should use source blocks:

,#+BEGIN_SRC org
,* Example Heading
This is example content.
,#+END_SRC

Keybinding references should use tables:

| Key           | Command                | Description          |
|---------------+------------------------+----------------------|
| Ctrl+C Ctrl+C | Execute block          | Run current block    |
| Tab           | Toggle fold            | Expand/collapse      |
