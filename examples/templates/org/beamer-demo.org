* Introduction
** Welcome
This is a regular frame at the frame level (=H:2=).

A *bold* word renders as =\alert{bold}= by default. Italics like /this/ and
~code~ stay the way the LaTeX backend would produce them.

Inline overlays via export snippets: place text @@beamer:<2->@@between
those markers@@beamer:<.->@@ and they appear only on the matching slide.

Special characters are escaped: 100\% & < > $ # _ ^ { }.

* Frame Options
** Action overlay on the frame
:PROPERTIES:
:BEAMER_act: <2->
:END:

This whole frame is wrapped in =\begin{frame}<2->=, so it doesn't appear
until the second overlay step of its enclosing slideshow chunk.

** Fragile is added automatically
A frame whose body contains verbatim-flavored content is automatically
promoted to =[fragile]=. No need to set it manually.

#+BEGIN_SRC python
def hello():
    print("hello, beamer")
#+END_SRC

** allowframebreaks
:PROPERTIES:
:BEAMER_opt: allowframebreaks
:END:

When a frame is too long for one slide, =allowframebreaks= lets Beamer
spill it across multiple slides. Auto-label is suppressed (Beamer does
not label split frames).

** Custom label and extra options
:PROPERTIES:
:BEAMER_opt: fragile,label=summary,t
:END:

Frames can carry any of: =fragile=, =plain=, =t=, =b=, =c=,
=allowframebreaks=, =label=, =shrink=, etc.

** A subtitle on the frame
:PROPERTIES:
:BEAMER_SUBTITLE: extra context
:END:

The =BEAMER_SUBTITLE= property fills in the second brace of
=\begin{frame}{title}{subtitle}=.

** A full-bleed frame
:PROPERTIES:
:BEAMER_env: fullframe
:END:

A frame with an empty title — useful for image-only or quote-only slides.

** Fragile guard for verbatim begin frame
A literal beamer frame opener that appears in =verbatim= text inside
the body is rewritten with an extra space so it does not close the
surrounding frame prematurely.

* Block Environments
** Default block
:PROPERTIES:
:BEAMER_env: block
:END:

The default environment for headlines deeper than the frame level. With
=H:2= we have to force =BEAMER_env: block= at level 2 to demo it
explicitly; otherwise level-3 children of a frame would be blocks
automatically.

* Source Code
** Python
A =minted= block is automatically marked fragile.

#+BEGIN_SRC python
import math

def gcd(a, b):
    """Greatest common divisor."""
    while b:
        a, b = b, a % b
    return a

print(gcd(48, 18))   # -> 6
#+END_SRC

* Math and References
** Inline and display math
Inline math like $\sin^2\theta + \cos^2\theta = 1$ works the same as in
the LaTeX backend.

\begin{equation}
\label{eq:euler}
e^{i\pi} + 1 = 0
\end{equation}

Reference Equation [[eq:euler]] above.

** Citations (commented out)
Citations use the standard org-cite syntax. The Beamer backend reuses
the LaTeX citation logic verbatim, so =#+CITE_EXPORT: bibtex= or
=biblatex= behaves the same way. Add a =#+BIBLIOGRAPHY:= keyword and
=\usepackage{natbib}= via =#+LATEX_HEADER:= to enable rendering.

* Speaker Notes and Repeats
** A frame with a separate note
:PROPERTIES:
:BEAMER_opt: label=funfact
:END:

Did you know Beamer was authored by Till Tantau?

** Speaker note for the previous frame
:PROPERTIES:
:BEAMER_env: note
:END:

This shows up on the speaker's monitor only. The headline title becomes
the note's heading.

** Speaker note without a heading
:PROPERTIES:
:BEAMER_env: noteNH
:END:

Body-only speaker note (=noteNH= drops the heading).

** Show the funfact frame again
:PROPERTIES:
:BEAMER_env: againframe
:BEAMER_ref: funfact
:BEAMER_act: <1->
:END:

This emits =\againframe<1->{funfact}=, replaying the labeled frame from
earlier.

* Heading Manipulation
** This heading does not appear
:PROPERTIES:
:BEAMER_env: ignoreheading
:END:

The headline is dropped, but its body content is rendered at the parent
scope. Useful for inserting raw LaTeX without an extra slide title.

** Inline raw LaTeX
You can also drop into raw LaTeX with an export block:

#+BEGIN_EXPORT beamer
\begin{tikzpicture}
  \draw[->, thick] (0,0) -- (2,1) node[right]{$\vec v$};
\end{tikzpicture}
#+END_EXPORT

Or with single-line passthroughs:

#+BEAMER: \pause
